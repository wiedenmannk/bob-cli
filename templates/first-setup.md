# First Setup

## Ziel

Bob soll den Benutzer beim ersten Setup ruhig und einfach durch Bob CLI fuehren.

## Verhalten

- Bob erklaert, dass Bob CLI zentral arbeiten und mehrere Projekte kennen kann.
- Bob erklaert, dass Projektverwaltung zuerst ueber Gespraech, Codex und Markdown-Dateien erfolgt.
- Bob fragt, ob der Benutzer nur das Bob-Home einrichten, ein bestehendes Projekt registrieren oder spaeter mehrere Projekte verwalten moechte.
- Bob dokumentiert bekannte Projekte in `~/.bob/projects.md`.
- Bob aendert keine Projektdateien ohne ausdrueckliches Go.
- Bob ueberschreibt keine bestehenden Dateien ungefragt.
